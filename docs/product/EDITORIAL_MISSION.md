# Editorial Mission

Authority Level: Product
Governed System: Editorial purpose and quality posture.
Describes: Both

## Scope
Defines editorial standards for historical content.

## Non-Scope
Does not define a complete source credibility matrix.

## Verified Implementation
Events require title, description, date, precision, and importance. Sources and tags are optional but relationship-backed when present. Timeline requests collect user suggestions for editorial review.

## Future Architecture
Editorial work should produce concise, sourced, canonical milestones with explicit uncertainty and review history.

## Dependencies
`src/server/validation/schemas.ts`, `docs/governance/*`.

# Import Batch Model

Authority Level: Factory Implementation Guidance
Governed System: Import batches as subordinate implementation-oriented Factory artifacts.
Describes: Both

## Scope
This document defines import batches as implementation-oriented artifacts beneath Factory objects and Publication Packages.

## Non-Scope
This document is not constitutional authority. It does not define Factory identity, Historical Library admission, or Publication Package doctrine.

## Product Identity
Import batches may support the Chronological Knowledge Factory, but they are not the Factory's canonical knowledge model.

## Constitutional Boundary
Factory constitutional authority belongs to `FACTORY_CONSTITUTION.md`. Import batches are subordinate to:
- Factory objects with permanent UUID identities.
- Historical Objects.
- Milestones.
- Timeline Views.
- Publication Packages.
- Feedback Packages.

## Publication Package Doctrine
An import batch may be one technical carrier or implementation artifact associated with a Publication Package. It must not replace the Publication Package concept.

## Feedback Package Doctrine
Import batches do not represent editorial amendments unless explicitly associated with a Feedback Package.

## Tiered Source Authority
Any import batch containing evidence must preserve source tier and provenance context required by Tiered Source Authority.

## Current Reality
No durable import batch table exists. Relationship recovery reports are durable; normal import results are returned to the caller.

## Future Architecture
Import batch documentation remains valid only as implementation guidance for moving Factory-approved content. It is not the source of Factory doctrine.

## Dependencies

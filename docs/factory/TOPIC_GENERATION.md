# Topic Generation

Authority Level: Factory Operational
Governed System: Topic candidates for chronological knowledge production.
Describes: Both

## Scope
This document governs topic candidate language inside Factory documentation.

## Non-Scope
This document does not implement topic generation or define content strategy.

## Product Identity
Topic generation serves TiMELiNES as a Chronological Knowledge Platform. A topic is not automatically a Timeline View; it may imply Historical Objects, Milestones, relationships, or narrative views.

## Historical Object Doctrine
Topic candidates may begin from Historical Objects such as Person, Institution, Place, Technology, Publication, Conflict, Movement, or Period.

## Milestone Doctrine
Topic candidates must be evaluated for Milestone production because Milestones are the canonical chronological knowledge unit.

## Timeline View Doctrine
A topic may become a Timeline View only when it can be expressed as a curated narrative view built from Milestones and Historical Objects.

## Publication Package Doctrine
Topic generation does not publish. Publication requires a Publication Package and Historical Library admission.

## Feedback Package Doctrine
Feedback Packages may create or refine topic candidates when Published Memory exposes gaps, disputes, amendments, or quality issues.

## Current Reality
Public users can submit timeline requests. Admins can update request statuses. These requests are current product inputs, not Factory topic objects.

## Future Architecture
Topic doctrine must preserve Historical Objects -> Milestones -> Timeline Views and must not equate every topic with a timeline.

WAVE_2A_LOCK

Status: LOCKED

Authority Level: Platform Capability Lock

Date: June 2026

⸻

Purpose

This document records the final locked decisions of Wave 2A.

Wave 2A introduced Historical Context as a first-class user experience capability within TiMELiNES.

The purpose of Wave 2A was not to create an entity platform, discovery platform, graph explorer, encyclopedia, or recommendation system.

The purpose of Wave 2A was to enrich chronology through contextual historical understanding.

⸻

Wave 2A Mission

Historical Context should help users understand:

* Who mattered
* What mattered
* Where it happened
* Why it mattered

without reducing the primacy of chronology.

Wave 2A exists to strengthen Milestones.

Wave 2A does not compete with Milestones.

Historical Context Doctrine

Historical Context is an enrichment layer.

Historical Context exists only to explain Milestones.

Historical Context is not a discovery mechanism.

Historical Context is not a graph surface.

Historical Context is not an encyclopedia surface.

Historical Context should answer:

Why did this Milestone matter?

Historical Context should not answer:

What other entities are related?

⸻

Participation Doctrine

Participation is the primary context unit.

Participation must be represented publicly as:

Object Name

Role

Meaning

Participation is not a tag.

Participation is not a relationship edge.

Participation is not metadata.

Participation is historical meaning.

⸻

Participation Priority Doctrine

Approved priority tiers:

1. PRIMARY
2. SUPPORTING
3. CONTEXT
4. BACKGROUND

Priority determines:

* visibility
* overflow behavior
* context ordering

Priority must always outrank grouping.

A higher-priority participation must never be hidden by a lower-priority participation because of object-type grouping.

⸻

Historical Object Doctrine

Historical Objects remain secondary surfaces.

Historical Objects exist because they are constitutional authorities.

Historical Objects do not exist to become the primary browsing model.

Object pages are contextual reference pages.

Object pages are not encyclopedia pages.

Object pages are not discovery destinations.

Object pages must always route users back toward chronology.

⸻

Required Historical Object Structure

Approved sections:

* Identity
* Summary
* Participation History
* Related Milestones
* Related Timelines

Additional sections require future architectural approval.

⸻

Context Grouping Doctrine

Approved context groups:

* People
* Institutions
* Places
* Technologies
* Publications

* Movements
* Periods

Group order is fixed and authoritative.

⸻

Context Density Doctrine

Historical Context must remain compact.

Chronology must remain visually dominant.

Historical Context must not overwhelm:

* milestone narrative
* chronology
* significance

Overflow must preserve editorial intent.

⸻

Product Identity Protection

Wave 2A must not change the identity of TiMELiNES.

TiMELiNES remains:

A Chronological Knowledge Platform.

It does not become:

* an entity platform
* an encyclopedia
* a graph explorer
* a recommendation system

Chronology remains the organizing principle of the platform.

Lessons Learned

Historical Objects strengthen chronology when introduced through Milestones.

Participation is the correct public context unit.

Historical Context is most effective when it explains significance rather than expands scope.

Object pages remain valuable only when they reinforce chronology.

Chronology-first navigation successfully prevents entity gravity.

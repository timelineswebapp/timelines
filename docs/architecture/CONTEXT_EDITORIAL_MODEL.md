Purpose

This document defines the editorial rules governing Historical Context within TiMELiNES.

It answers:

* Which participants appear?
* Which participants are hidden?
* How are participants prioritized?
* How is context grouped?
* How much context should be visible?
* How should participation be expressed?

This document governs all Historical Context experiences.

⸻

Constitutional Alignment

Historical Context exists to support Milestones.

Historical Context does not replace Milestones.

Historical Context does not replace Timeline Views.

Historical Context exists to help users understand chronology.

The governing hierarchy remains:

Timeline View
        ↓
Milestone
        ↓
Historical Context
        ↓
Historical Object

⸻

Historical Context Definition

Historical Context is the user-facing representation of:

Historical Object
        ↓
Participation
        ↓
Milestone

Historical Context explains:

* who mattered
* what mattered
* where it happened
* which institutions mattered
* which technologies mattered
* which movements mattered
* why they mattered

Historical Context is explanatory.

Historical Context is not encyclopedic.

⸻

Participation-Centered Model

Participation is the primary context unit.

Historical Objects are not displayed alone.

Historical Objects are displayed through Participation.

Users should learn:

Why this object mattered here.

before learning:

What this object is.

⸻

Participation Display Model

Canonical display format:

Object Name
Role
Meaning

Example:

NASA
Program Operator
Led and managed the Apollo 11 mission.

Example:

Neil Armstrong
Commander
First human to walk on the Moon during Apollo 11.

Rules:

* Role is always visible.
* Meaning is visible for priority participants.
* Meaning must be concise.
* Meaning must explain historical significance.
* Meaning must not repeat the object summary.

⸻

Context Priority Model

Every Participation must belong to one editorial priority tier.

⸻

Tier 1 — Primary Participant

Definition:

The milestone cannot be understood without this participant.

Examples:

Apollo 11:

* Neil Armstrong
* NASA

Fall of Constantinople:

* Mehmed II
* Constantinople

Display:

Always visible.

⸻

Tier 2 — Supporting Participant

Definition:

Materially contributes to understanding the milestone.

Examples:

Apollo 11:

* Buzz Aldrin
* Saturn V

Display:

Visible by default.

⸻

Tier 3 — Context Participant

Definition:

Provides useful additional context.

Examples:

Mission Control
Apollo Guidance Computer

Display:

May be collapsed.

⸻

Tier 4 — Background Participant

Definition:

Provides broader historical framing.

Examples:

Space Race
Cold War

Display:

Collapsed by default.

⸻

Context Grouping Model

Historical Context is grouped by Historical Object type.

Canonical order:

1. People
2. Institutions
3. Places
4. Technologies
5. Publications
6. Conflicts
7. Movements
8. Periods

This order is platform-wide.

⸻

Context Density Rules

Editorial target:

Approximately six visible context items.

This is not a hard limit.

Editorial judgment remains authoritative.

Rules:

* Show only context necessary for understanding.
* Avoid exhaustive participant lists.
* Avoid repetition.
* Avoid historical trivia.
* Prioritize significance over completeness.

⸻

Context Overflow Rules

Overflow must preserve:

* priority order
* object type grouping

Overflow should be organized as:

More People
More Technologies
More Context

Avoid:

Show 27 More

The user should understand what additional context exists before expanding.

⸻

Milestone Context Placement

Historical Context appears after:

* chronology
* milestone narrative
* milestone significance

Historical Context appears before:

* sources
* metadata
* administrative information

Canonical milestone order:

1. Chronology
2. Title
3. Narrative
4. Significance
5. Historical Context
6. Sources

⸻

Historical Object Page Scope

Historical Object pages are secondary surfaces.

Required sections:

* Identity
* Summary
* Participation History
* Related Milestones
* Related Timeline Views

⸻

Historical Object Page Exclusions

Do not include:

* Object-to-object relationships
* Graph exploration
* Recommendations
* Discovery feeds
* AI interpretation
* Long-form biographies
* Encyclopedia-style essays

Wave 2 is not creating a historical encyclopedia.

⸻

Editorial Success Criteria

Historical Context succeeds when:

* users understand who mattered
* users understand what mattered
* users understand why it mattered
* chronology remains dominant
* narrative remains dominant

Historical Context fails when:

* milestones become secondary
* context overwhelms chronology
* object pages become the primary destination
* TiMELiNES feels like an entity directory
* TiMELiNES feels like a graph explorer

⸻

Final Editorial Authority Statement

Historical Context exists to enrich Milestones.

Participation provides meaning.

Historical Objects provide context.

Milestones provide chronology.

Timeline Views provide narrative.

Chronology remains the organizing principle of TiMELiNES.

WAVE_2_PRODUCT_MISSION.md

Status

LOCKED PRODUCT AUTHORITY

⸻

Purpose

This document defines the mission, boundaries, objectives, and success criteria of Wave 2.

Wave 2 is the first public experience wave built on top of the Historical Authority Foundation established in Wave 1.

Wave 2 does not create new historical authority.

Wave 2 exposes existing historical authority to users.

⸻

Wave Name

Historical Context Experience

⸻

Mission Statement

Wave 2 exists to enrich Milestones with Historical Context while preserving Milestones and Timeline Views as the primary user experience of TiMELiNES.

⸻

Constitutional Alignment

Wave 2 must remain aligned with:

Historical Object
        ↓
Participation
        ↓
Milestone
        ↓
Timeline View

and

Historical Objects provide context.
Participation provides meaning.
Milestones provide chronology.
Timeline Views provide narrative.

Wave 2 must expose this model without changing it.

⸻

Product Identity

TiMELiNES remains a:

Chronological Knowledge Platform.

TiMELiNES is not:

* an entity directory
* a knowledge graph explorer
* a historical encyclopedia
* a people database
* an object catalog

Chronology remains the organizing principle of the platform.

⸻

Primary Objective

Enable users to better understand Milestones through Historical Context.

Wave 2 is successful when users can answer:

* Who participated?
* What participated?
* Where did it happen?
* Which institutions mattered?
* Which technologies mattered?
* Which movements mattered?
* Why was this milestone historically significant?

without leaving the Milestone experience.

⸻

Experience Hierarchy

The information hierarchy of TiMELiNES remains:

Timeline View
        ↓
Milestone
        ↓
Historical Context
        ↓
Historical Object

Historical Context supports Milestones.

Historical Objects support Historical Context.

Milestones remain the center of the experience.

⸻

Core User Journey

Primary navigation remains:

Timeline
        ↓
Milestone

Wave 2 introduces:

Timeline
        ↓
Milestone
        ↓
Historical Context

Historical Objects become accessible through context.

Historical Objects do not become the primary navigation model.

⸻

Historical Context

Historical Context is the first public manifestation of Historical Object Authority and Participation Authority.

Historical Context may include:

People

Examples:

* Neil Armstrong
* Napoleon Bonaparte
* Mahatma Gandhi

Institutions

Examples:

* NASA
* United Nations
* Roman Senate

Places

Examples:

* Sea of Tranquility
* Constantinople
* Berlin

Technologies

Examples:

* Saturn V
* Printing Press
* World Wide Web

Movements

Examples:

* Renaissance
* Civil Rights Movement
* Industrial Revolution

Conflicts

Examples:

* World War II
* Peloponnesian War

Periods

Examples:

* Space Race
* Victorian Era

Historical Context is grouped by Historical Object type.

⸻

Participation Visibility

Participation becomes visible for the first time in Wave 2.

Participation must communicate:

* why the object matters
* how the object relates to the milestone
* the object’s historical role

Examples:

NASA
Role: Program Operator
Neil Armstrong
Role: Commander
Saturn V
Role: Launch Vehicle

Participation is not displayed as a technical relationship.

Participation is displayed as historical meaning.

⸻

Milestone Context Experience

Milestones remain the primary knowledge surface.

Historical Context must appear as supporting information.

Historical Context must never overwhelm:

* chronology
* milestone narrative
* milestone significance

The Milestone remains the dominant content surface.

⸻

Historical Object Pages

Historical Object pages are secondary surfaces.

They exist because Historical Objects are authoritative entities.

Historical Object pages must remain lightweight.

Minimum objectives:

* Identity
* Summary
* Participation History
* Related Milestones
* Related Timeline Views

Historical Object pages are not the primary destination of the platform.

⸻

Context Navigation

Users may navigate between:

Historical Object
        ↔
Milestone
        ↔
Timeline View

Navigation exists to improve understanding of chronology.

Navigation does not exist to create a graph exploration product.

⸻

Editorial Principles

Principle 1

Chronology first.

⸻

Principle 2

Context supports chronology.

⸻

Principle 3

Narrative remains timeline-driven.

⸻

Principle 4

Historical Objects support understanding.

⸻

Principle 5

Participation explains meaning.

⸻

Principle 6

Complexity must not exceed educational value.

⸻

Explicit Non-Scope

Wave 2 must not introduce:

* Knowledge Graph Explorer
* Object-to-Object Relationships
* Recommendations
* AI Historical Interpretation
* Semantic Search
* Historical Object Search
* Historical Object Discovery Hub
* Historical Object Homepage Sections
* Factory Integration
* Publication Package UX
* Feedback Package UX

These belong to future waves.

⸻

Success Criteria

Wave 2 is successful when:

* Historical Context becomes visible.
* Participation becomes understandable.
* Milestones become richer.
* Timeline Views become more informative.
* Historical Objects become navigable.
* Chronology remains dominant.
* Narrative remains dominant.
* Users understand history better without increased complexity.

⸻

Failure Conditions

Wave 2 fails if:

* Historical Objects become the primary experience.
* Timelines become secondary.
* Milestones become secondary.
* Users experience TiMELiNES as an entity directory.
* Users experience TiMELiNES as a graph explorer.
* Historical Context overwhelms chronology.
* Participation becomes technical rather than meaningful.

⸻

Wave 2 Authority Statement

Wave 2 does not exist to expose Historical Objects.

Wave 2 exists to expose Historical Context.

Historical Objects provide context.

Participation provides meaning.

Milestones provide chronology.

Timeline Views provide narrative.

Chronology remains the organizing principle of TiMELiNES.
